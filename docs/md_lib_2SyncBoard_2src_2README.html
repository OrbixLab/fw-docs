SyncBoard Folder Structure

This folder contains the core framework and modules for the Orbix project. The architecture is organized to separate hardware abstraction, plugin management, and utility code for maintainability and scalability.

Folder Structure

  • core/
    Contains the main framework logic, including the core event loop, plugin manager, and system initialization.
  • hardware/
    Hardware abstraction layer. Includes drivers and interfaces for peripherals such as LEDs, buttons, battery management, NFC readers, and display modules.
  • plugins/
    Contains all plugin implementations. Each plugin adds a specific feature or hardware support (e.g., LED management, battery monitoring, communication modules).
  • events/
    Event system definitions and handlers. Used for communication between plugins and the core system.
  • utils/
    Utility functions and helper classes used throughout the framework.
  • include/
    Public header files for the SyncBoard framework, exposing APIs for use by games and other modules.

File Overview

  • SyncBoard.h
    Main include file for the SyncBoard framework.
  • Plugin.h
    Base class for all plugins.
  • EventHandler.h
    Event system interface.
  • Config.h
    Configuration definitions and macros.

How to Extend

To add new hardware support or features:

  1. Implement a new plugin in the plugins/ directory.
  2. Register the plugin in the main application.
  3. Use the event system for communication between plugins and the core.

For more details, refer to the inline documentation in each source file.
